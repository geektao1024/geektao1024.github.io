# geektao.github.io

`GeekTao 工作室` 的 GitHub Pages 主页仓库。

## 重要说明

你当前创建的仓库名是 `geektao.github.io`，仓库所有者是 `geektao1024`。

按 GitHub Pages 官方规则：

- 用户站点仓库必须命名为 `<owner>.github.io`
- 也就是如果你想要根地址 `https://geektao1024.github.io/`
- 仓库名应该是 `geektao1024.github.io`

所以当前这个仓库默认更像“项目站点”，发布地址通常会是：

- `https://geektao1024.github.io/geektao.github.io/`

如果你就是想做成类似 `rokcso.github.io` 这种根级 GitHub 二级域名站点，建议把仓库改名为：

- `geektao1024.github.io`

## 当前仓库包含

- `index.html`：GeekTao 工作室首页
- `styles.css`：全站样式
- `404.html`：自定义 404 页面
- `tools/starter/index.html`：示例工具页
- `.nojekyll`：禁用 GitHub Pages 默认 Jekyll 构建

## 新增工具的推荐结构

1. 在 `tools/` 下创建目录，例如 `tools/json-viewer/`
2. 把工具页面和静态资源放进该目录
3. 在 `index.html` 的站点或工具区补一个入口链接
4. 推送到 `main`

## GitHub Pages 设置

在仓库设置里确认：

1. `Settings`
2. `Pages`
3. `Build and deployment`
4. `Source` 选择 `Deploy from a branch`
5. `Branch` 选择 `main`
6. Folder 选择 `/ (root)`

## 站点定位

这个首页已经调整成类似 `rokcso.github.io` 的独立工作室风格，用来展示：

- GeekTao 工作室品牌
- 现有网站矩阵
- 后续静态工具入口
- GitHub Pages 路径说明
